CATEGORY 1: HOUSEHOLD REPAIR & MAINTENANCE
================================================================================
Platform : HomeFix
Currency : INR (₹)
Version  : 2.0 — Deep Algorithm Spec
================================================================================

GLOBAL CONSTANTS (apply to every service in this category)
------------------------------------------------------------
CITY_TIER_MULTIPLIER:
  T1 = 1.30   → Mumbai, Delhi NCR, Bengaluru, Chennai, Hyderabad, Pune, Kolkata
  T2 = 1.10   → Jaipur, Surat, Lucknow, Nagpur, Indore, Bhopal, Coimbatore, Kochi
  T3 = 1.00   → all remaining towns and cities

GEO_RADIUS:
  primary   = 3 km   (all services except Welder)
  secondary = 5 km   (fallback when < 2 pros available in primary)
  welder    = 5 km   (portable welding kit required; wider base needed)

TRUST_SCORE:
  formula = (avg_rating * 0.40)
           + (completion_rate * 0.30)
           + (response_time_score * 0.20)
           + (repeat_hire_rate * 0.10)
  range   = 0.0 – 5.0
  minimum to appear in results = 3.5

EMERGENCY_SURGE = 1.5×  (applied to labour only; never to parts or visit fee)
EMERGENCY_AUTO_FLAGS:
  - issue_type == regulator_leak     (Gas Stove)
  - issue_type == pipe_burst         (Plumber)
  - issue_type == no_power_whole_flat (Electrician)

OTP SYSTEM (universal across all services):
  start_otp : 6-digit code → generated at booking → sent to user's phone
              → pro enters on arrival → booking clock starts
              → billable time begins from this moment
  end_otp   : 6-digit code → generated when pro taps "job done"
              → sent to user → user enters → billing stops
              → escrow release triggered automatically
  wrong OTP 3× by pro → platform agent auto-alerted; job paused

ESCROW RULES:
  deposit collected at  : booking confirmation
  balance held until    : end_otp confirmed OR mutual cancellation
  auto_release after    : 48 hours post end_otp (if no dispute raised)
  dispute_window        : 48 hours after end_otp
  cancellation refund   : 100% if cancelled before start_otp
                          0% if cancelled after start_otp

MANDATORY PHOTO MILESTONES (all services):
  before_photo : uploaded by pro before any work begins (proof of condition)
  parts_photo  : each part purchased → receipt + item photo (logged in booking)
  progress_photo: end of each day for multi-day jobs
  after_photo  : uploaded by pro on completion → triggers end_otp eligibility

DIGITAL INVOICE FIELDS (all services):
  booking_id, pro_name, start_time, end_time, duration_mins,
  labour_charge, parts_itemised[ {name, qty, unit_price} ],
  visit_fee, surge_applied (bool), city_tier, total,
  payment_mode, otp_start_verified (bool), otp_end_verified (bool)

1.1  ELECTRICIAN
================================================================================

INPUTS:
  job_type   : fan_install_repair | switch_socket_repair | wiring_work
               | new_socket_point | mcb_fusebox
  location   : GPS lat/lng + address text
  urgency    : normal | emergency
  city_tier  : T1 | T2 | T3  (auto-detected from GPS; user can override)

COMPLEXITY TIERS:
  T1 — Simple   : fan_install_repair, switch_socket_repair, new_socket_point
  T2 — Moderate : wiring_work, mcb_fusebox
  T3 — Advanced : full_rewiring, distribution_board  (requires licensed cert upload)

BASE LABOUR PRICE TABLE (T3 city, before multiplier):
  job_type                  T3_base (₹)   notes
  ──────────────────────────────────────────────────────────
  fan_install_repair           170         ceiling / exhaust / table fan
  switch_socket_repair          90         single point; standard face plate
  wiring_work                  450         per circuit; conduit included
  new_socket_point             120         point + conduit run up to 2 m
  mcb_fusebox                  280         MCB replacement or fuse box work
  ──────────────────────────────────────────────────────────
  final_base = T3_base × CITY_TIER_MULTIPLIER[city_tier]

VISIT FEE (flat; always added; not part of estimate range):
  T1 city → ₹100     T2 city → ₹75     T3 city → ₹50

ALGORITHM:
  Step 1   User selects job_type from dropdown.
           system looks up: complexity_tier = COMPLEXITY_MAP[job_type]
           display to user: "This is a [T1 / T2] complexity job."

  Step 2   System reads GPS → resolves city_tier.
           IF city_tier ambiguous (border town): show manual picker.

  Step 3   base_price = PRICE_TABLE[job_type]['T3'] × CITY_TIER_MULTIPLIER[city_tier]
           estimate_lo = round(base_price × 0.88)
           estimate_hi = round(base_price × 1.18)
           Display range now, BEFORE urgency applied.

  Step 4   Urgency check:
           IF urgency == EMERGENCY:
             surge_factor = 1.5
             base_price  *= 1.5
             estimate_lo  = round(base_price × 0.88)
             estimate_hi  = round(base_price × 1.18)
             push alert: "🚨 Emergency surge 1.5× applied to labour."
           ELSE:
             surge_factor = 1.0  (no change)

  Step 5   Geo-query:
           candidates = pros WHERE
             profession == 'electrician'
             AND distance(pro, user) <= GEO_RADIUS.primary
             AND verified_license == true
             AND status == 'available'
             AND trust_score >= 3.5
           IF len(candidates) < 2 → expand to GEO_RADIUS.secondary
           IF len(candidates) == 0 → show "No pros available now. Schedule for tomorrow?"

  Step 6   Rank candidates:
           FOR each pro: rank_score = TRUST_SCORE formula
           sort DESC by rank_score
           show user: top 3 pros — name, photo, trust_score, distance, total_jobs_done

  Step 7   User selects pro → booking confirmed.
           deposit = ₹0  (electrician jobs; no advance)
           escrow_hold = estimate_lo  (platform wallet)

  Step 8   OTP flow:
           start_otp sent to user → pro enters on arrival
           clock_start = arrival timestamp
           pro uploads before_photo (mandatory; job cannot proceed without it)

  Step 9   During job — parts logging:
           FOR each part used (wire, socket, MCB, fuse, tape…):
             pro enters: part_name + unit_price + quantity
             receipt_photo uploaded
             part_line_item appended to live invoice

  Step 10  On-site quote revision:
           IF actual job scope > original job_type:
             pro raises revision request in-app
             user sees: "Additional work found: [description]. Extra ₹[amount]. Approve?"
             user: APPROVE → work continues
             user: REJECT  → only original scope completed; revised invoice

  Step 11  Completion:
           pro uploads after_photo
           pro taps "Job done" → end_otp sent to user
           user enters end_otp → job status = 'completed'
           clock_stop = timestamp

  Step 12  Invoice auto-generated:
           labour_charge = base_price (agreed or revised)
           parts_total   = SUM(part_line_items)
           visit_fee     = VISIT_FEE[city_tier]
           IF surge: label "Emergency 1.5×" on invoice line
           total = labour_charge + parts_total + visit_fee
           PDF emailed to user + available in-app

  Step 13  Payment:
           UPI → QR generated in-app; escrow released on scan confirmation
           Cash → pro marks paid; platform confirms; escrow released
           Rating prompt: 1–5 stars + optional text (shown 10 min after payment)

1.2  PLUMBER
================================================================================

INPUTS:
  issue_type  : leak_repair | tap_repair_replace | commode_toilet_issue
                | geyser_repair | pipe_burst
  photos      : 1–3 images of problem area (mandatory before quote is sent)
  location    : GPS lat/lng + address
  urgency     : normal | emergency
                (pipe_burst auto-forces urgency = EMERGENCY)

BASE PRICE RANGES (T3, before city multiplier):
  issue_type               T3 lo (₹)    T3 hi (₹)    notes
  ──────────────────────────────────────────────────────────────────────
  leak_repair                 250           550        joint, pipe, fitting
  tap_repair_replace          180           380        part cost billed extra
  commode_toilet_issue        350           750        flush valve, seat, wax ring
  geyser_repair               500          1100        element, thermostat, valve
  pipe_burst (EMERGENCY)      700          1500        + 1.5× surge on labour
  ──────────────────────────────────────────────────────────────────────
  displayed_lo = T3_lo × CITY_TIER_MULTIPLIER
  displayed_hi = T3_hi × CITY_TIER_MULTIPLIER

DEPOSIT MODEL:
  deposit_pct  = 20%
  deposit_base = displayed_lo  (lower bound of quote accepted by user)
  deposit_amount = round(accepted_quote × 0.20)
  escrow holds: accepted_quote − deposit_amount

ALGORITHM:
  Step 1   User selects issue_type.
           IF issue_type == 'pipe_burst':
             urgency = EMERGENCY  (forced; cannot be overridden)
             show red alert: "Pipe burst detected. Priority dispatch activated.
                              Turn off main water valve now if possible."

  Step 2   Photo upload:
           required_min = 1,  recommended = 3
           IF no photo: block progression with message:
             "Photo helps plumber quote accurately. Attach at least 1 photo to continue."
           photos stored against booking object.

  Step 3   Remote quote broadcast:
           broadcast job_details + photos to all plumbers within GEO_RADIUS.primary
           quote_window = 15 minutes
           plumbers enter: { quoted_price, ETA_minutes, note_to_user }
           IF 0 responses after 15 min: expand radius + rebroadcast; notify user.

  Step 4   User receives quotes:
           display each: plumber_name, trust_score, quoted_price, ETA
           user action:
             ACCEPT  → proceed to deposit payment
             REJECT  → re-broadcast (once more; then manual assignment by platform)
             COUNTER → open chat; plumber can revise quote once

  Step 5   Booking confirmed (after accept):
           deposit_amount = round(accepted_quote × 0.20)
           UPI payment collected: deposit_amount
           slot confirmed; plumber status → 'en_route'
           SMS to user: "Plumber [name] on the way. ETA [X] min."

  Step 6   On-site arrival:
           plumber enters start_otp
           before_photo taken and uploaded (mandatory)

  Step 7   On-site quote revision (if extra work found):
           plumber raises revision_request:
             { description: text, extra_amount: ₹N, photo_of_extra_work }
           user notification: "Plumber found [desc]. Extra ₹[N] needed. Approve?"
           IF APPROVE: revised total locked; work continues
           IF DISPUTE:
             platform_agent joins 3-way chat
             agent views: original photos, revision photo, chat log
             resolution options: partial_approval | refund_deposit | cancel

  Step 8   Job completion:
           plumber uploads after_photo
           plumber taps "Done" → end_otp sent to user
           user enters end_otp → status = 'completed'

  Step 9   Balance payment:
           balance = final_agreed_price − deposit_amount
           user pays balance (UPI or cash)
           deposit escrow released to plumber
           total_paid_by_user = deposit + balance

  Step 10  Service log entry:
           { booking_id, issue_type, plumber_id, original_quote,
             revision_amount, start_time, end_time, duration_mins,
             photos[ ], parts_used[ ], rating }

1.3  CARPENTER
================================================================================

INPUTS:
  furniture_type      : chair | table | wardrobe | door | drawer | bed | sofa_frame
                        | cupboard | almirah | bookshelf
  problem_description : free text (max 300 chars)
  photos              : 1–5 images
  dimensions          : L × W × H in cm (optional; carpenter may measure on-site)
  quote_preference    : per_job | per_hour | let_carpenter_decide

BASE PRICE RANGES (T3):
  work_category              T3 lo (₹)   T3 hi (₹)   billing_unit
  ────────────────────────────────────────────────────────────────────
  hinge_joint_repair            200          450       per job
  door_drawer_fix               350          650       per job
  polish_refinish               700         1400       per piece
  furniture_repair              500         1000       per job
  full_refurbish               1400         3200       per piece
  ────────────────────────────────────────────────────────────────────
  PER-HOUR rate (T3): ₹250/hr × CITY_TIER_MULTIPLIER

REPAIR vs REFURBISH CLASSIFICATION:
  system scans problem_description for keywords:
    repair_signals    : [broken, cracked, hinge, stuck, loose, wobble, chip, scratch]
    refurbish_signals : [colour gone, full polish, termite, water damage, makeover,
                         full restore, complete redo]
  IF count(refurbish_signals) > count(repair_signals):
    show comparison card:
      Option A — Repair    : ₹[repair_lo]–₹[repair_hi]   (targeted fix only)
      Option B — Refurbish : ₹[refurb_lo]–₹[refurb_hi]  (full restoration)
    user picks one; selection locked before carpenter is contacted

ALGORITHM:
  Step 1   User uploads photos + types problem_description.
           System runs keyword scan → suggests repair vs refurbish.
           User confirms chosen option.

  Step 2   Quote type selection:
           IF quote_preference == per_job:
             carpenter sends fixed_price quote (locked at booking; no clock)
           IF quote_preference == per_hour:
             carpenter sends hourly_rate + estimated_hours
             → show user: "~₹[hrs × rate] estimate; actual billed by clock"
           IF let_carpenter_decide:
             carpenter chooses and declares in their quote message

  Step 3   Carpenter views booking:
           sees { furniture_type, description, photos, option_chosen, dimensions_if_given }
           Carpenter responds:
           {
             quote_type       : per_job | per_hour,
             quoted_price     : ₹N  (or hourly_rate if per_hour),
             estimated_hours  : H   (only for per_hour),
             material_note    : "hinges / polish / wood needed — cost extra",
             available_slot   : date + time range
           }
           User: ACCEPT | COUNTER | REJECT

  Step 4A  IF per_job quote accepted:
           fixed_price locked
           deposit = ₹0  (no advance for carpenter)
           start_otp begins job (clock is informational only; payment is fixed)

  Step 4B  IF per_hour quote accepted:
           hourly_rate locked
           start_otp: clock_start = timestamp
           end_otp  : clock_stop  = timestamp
           billed_hours = ceil((clock_stop − clock_start) / 1800) × 0.5
             (rounded up to nearest 30 min)
           labour_charge = billed_hours × hourly_rate × CITY_TIER_MULTIPLIER

  Step 5   Material procurement:
           IF carpenter buys wood, hinges, polish, hardware:
             each item → name + qty + price + receipt_photo
             → appended as line items in live invoice
           IF single material purchase > ₹500:
             user pre-approval required in-app before purchase

  Step 6   Completion:
           after_photo uploaded (shows repaired / restored item clearly)
           user inspects in-person → enters end_otp if satisfied
           IF user rejects quality:
             platform review triggered; carpenter must redo within 2 days at no charge

  Step 7   Invoice:
           {
             labour   : fixed_price  (per_job)
                        OR hours_clocked × hourly_rate  (per_hour),
             materials: [ {item, qty, price} … ],
             visit_fee: ₹0  (included in carpenter's labour),
             total    : labour + materials_total
           }

1.4  PAINTER
================================================================================

INPUTS:
  room_count        : integer 1–20
  rooms[ ]          : each room → { room_name, length_ft, width_ft, height_ft }
  paint_type        : distemper | emulsion | enamel | texture
  surface_condition : new_plaster | old_repaint | damaged_patch
  colour_choice     : shade selected from in-app colour palette
  city_tier         : T1 | T2 | T3

PAINT RATE TABLE (₹ per sq.ft, T3 city, before multiplier):
  paint_type     rate_per_sqft   primer_per_sqft   standard_coats
  ─────────────────────────────────────────────────────────────────
  distemper           8               2            1 primer + 2 paint
  emulsion           14               3            1 primer + 2–3 paint
  enamel             18               3            1 primer + 2 paint
  texture            26               4            base coat + texture
  ─────────────────────────────────────────────────────────────────
  final_rate    = rate_per_sqft   × CITY_TIER_MULTIPLIER
  final_primer  = primer_per_sqft × CITY_TIER_MULTIPLIER

SURFACE CONDITION MULTIPLIER (applied to labour rate only; not to primer):
  new_plaster   → 1.00   (clean surface; no extra prep)
  old_repaint   → 1.10   (scraping + sanding adds ~10% labour)
  damaged_patch → 1.30   (crack filling + POP putty adds ~30% labour)

SQFT FORMULA (walls only; standard for painting quotes):
  wall_sqft_per_room = 2 × height_ft × (length_ft + width_ft)
  total_wall_sqft    = SUM(wall_sqft_per_room for all rooms)
  NOTE: doors/windows NOT deducted at estimate stage;
        painter deducts on-site if user requests (~5–8% typical)

COLOUR PALETTE TOOL:
  8 families: Arctic White | Warm Cream | Sage Green | Ocean Blue
              Terracotta | Charcoal | Dusty Rose | Lavender
  Each family: 4 sub-shades (light → deep)
  colour_code (hex) saved to booking; shown to painter before visit

ALGORITHM:
  Step 1   User enters room_count.
           FOR room i in 1..room_count:
             prompt: room_name, length_ft, width_ft, height_ft
             wall_sqft[i] = 2 × height_ft × (length_ft + width_ft)
           total_sqft = SUM(wall_sqft)
           display: "Total paintable wall area: [total_sqft] sq.ft"

  Step 2   paint_type selected.
           labour_rate  = PAINT_TABLE[paint_type].rate_per_sqft   × CITY_MULT
           primer_rate  = PAINT_TABLE[paint_type].primer_per_sqft × CITY_MULT

  Step 3   surface_condition selected.
           surface_mult          = SURFACE_MULTIPLIER[surface_condition]
           effective_labour_rate = labour_rate × surface_mult

  Step 4   auto_estimate:
           labour_cost      = total_sqft × effective_labour_rate
           primer_cost      = total_sqft × primer_rate
           raw_estimate     = labour_cost + primer_cost
           auto_estimate_lo = round(raw_estimate × 0.90)
           auto_estimate_hi = round(raw_estimate × 1.15)
           NOTE: paint tin cost NOT in estimate;
                 painter buys tins → user approves → added to invoice.

  Step 5   Colour selection shown.
           User picks shade → colour_code stored in booking_object.
           Preview: coloured 300 × 180 px wall swatch rendered in-app.

  Step 6   Painter assigned; painter receives:
           rooms[ ] + total_sqft + paint_type + surface_condition +
           colour_code + auto_estimate_range + photos_if_any.
           Painter may adjust estimate ≤ 10% after site visit without approval.
           IF adjustment > 10%: user approval required before work starts.

  Step 7   Work schedule:
           approx_daily_capacity = 300 sqft (1 coat for avg room size)
           estimated_days        = ceil(total_sqft / 300)
           END OF EACH DAY:
             painter uploads progress_photo + area_completed_sqft (running total)
             running_invoice updated: area_done × rate

  Step 8   Completion:
           final_photo uploaded by painter.
           User inspects (in-app checklist):
             [ ] No drips or streaks
             [ ] Patches covered uniformly
             [ ] Colour matches chosen shade
             [ ] Edges (near ceiling/floor) clean
           IF all passed → end_otp → balance payment released.
           IF defect found: painter redoes flagged area within 48 hrs at no charge.

  Step 9   Invoice:
           {
             rooms[ ], total_sqft_painted, paint_type, surface_condition,
             colour_code, paint_tins_purchased[ ], labour_per_sqft_rate,
             days_worked, labour_total, primer_cost, material_cost, total
           }

1.5  MASON
================================================================================

INPUTS:
  job_type        : tile_repair | plaster_patch | wall_crack_fill | boundary_wall
                    | waterproofing | flooring_new
  job_description : free text (max 500 chars)
  photos          : 2–10 images (problem area + any measurements visible)
  area_sqft       : user's approximate estimate (can be rough)
  urgency         : normal | urgent

BASE PRICE RANGES (T3, per sqft unless noted):
  job_type               T3 lo    T3 hi    unit
  ────────────────────────────────────────────────────────
  tile_repair               40       80    per sqft
  plaster_patch             25       55    per sqft
  wall_crack_fill          150      350    per crack (flat)
  boundary_wall (brick)    600      900    per running foot
  waterproofing             70      130    per sqft
  flooring_new              60      120    per sqft (labour only)
  ────────────────────────────────────────────────────────

BIDDING ENGINE:
  bidding_window = 2 hours  (from job post time)

  FOR each bid B:
    price_competitiveness[B] = (max_bid − B.price) / (max_bid − min_bid)
      (= 1.0 for lowest bidder; 0.0 for highest bidder)
    ranking_score[B] = (trust_score[B] × 0.50) + (price_competitiveness[B] × 0.50)
  sort bids DESC by ranking_score
  present top 3 to user with: name, trust_score, bid_price, days_est, score

PAYMENT STRUCTURE:
  advance              = 30% of accepted_bid  (collected at booking)
  escrow               = 60% of accepted_bid  (released after end_otp)
  punch_list_holdback  = 10% of accepted_bid  (released 7 days post-completion)

ALGORITHM:
  Step 1   User posts job: job_type + description + photos + area_sqft.
           System tag-classifies post; area_sqft shown as "approximate" to masons.

  Step 2   Platform broadcasts to all masons within GEO_RADIUS.primary.
           bidding_window_start = NOW()
           bidding_window_end   = NOW() + 7200 s
           Masons submit bids:
           {
             mason_id, quoted_price, estimated_days,
             material_included (bool), note_to_user
           }
           IF material_included == false: platform appends "(Materials billed separately)"

  Step 3   After bidding_window ends (or ≥5 bids received early):
           ranking_scores computed for all bids
           top_3 = sort(bids, ranking_score DESC)[:3]
           user sees top 3 side-by-side:
             mason name, trust_score, bid_price, days, ranking_score, material note

  Step 4   User selects bid.
           advance = round(selected_bid.price × 0.30)
           UPI payment collected.
           Mason status → 'assigned'.

  Step 5   Job start (start_otp):
           before_photo[ ] mandatory (all affected areas).

  Step 6   Material procurement (mason logs each purchase):
           item_name + qty + shop_bill_photo
           IF single_purchase > ₹1,000: user in-app approval required.

  Step 7   Progress tracking (jobs > 1 day):
           EACH DAY END:
             mason uploads progress_photo[ ]
             marks area_completed_sqft (running total visible to user in-app)

  Step 8   Punch-list (end of final day, before payment):
           user completes digital checklist:
             [ ] Tiles aligned and grouted with no hollow sounds
             [ ] Plastered surface smooth — no visible patches or cracks
             [ ] Cracks filled flush with wall surface
             [ ] Curing completed (applicable for cement work)
             [ ] Area cleaned of debris
           FOR each FAILED item:
             mason must redo within agreed window (default 2 working days)
           WHEN all items PASSED:
             user enters end_otp
             escrow 60% released to mason
             punch_list_holdback 10% released after 7-day defect period

  Step 9   Invoice:
           {
             job_type, area_sqft_actual, quoted_price, material_costs[ ],
             days_worked, advance_paid, escrow_released,
             holdback_released_date, punchlist_items[ ]
           }

1.6  WELDER
================================================================================

INPUTS:
  item_type           : gate | grill | railing | staircase | tank | furniture_frame
                        | shed_frame | window_frame
  job_type            : new_fabrication | repair_existing
  material_preference : mild_steel | stainless_steel | aluminium | wrought_iron
  photos              : 2–8 images (existing item or reference design)
  site_dimensions     : optional at booking; welder measures on-site if fabrication

MATERIAL COST REFERENCE (₹ per kg, approximate market rates):
  mild_steel        ₹ 65 – 80
  stainless_steel   ₹180 – 240
  aluminium         ₹160 – 200
  wrought_iron      ₹ 90 – 110
  (actual rate quoted by welder from current market at booking time)

LABOUR RATES (T3, ₹ per hour):
  repair_work    : ₹350 – 500
  fabrication    : ₹450 – 650  (skilled; design-to-execution)

ALGORITHM:
  Step 1   User uploads photos + selects item_type + job_type + material_preference.
           IF job_type == new_fabrication:
             system prompts: "Share a reference image or sketch for your design."
             design_photo stored against booking.

  Step 2   Geo-query welders within 5 km.
           Filter: has_portable_kit == true
                   AND skills include material_preference
           Rank by trust_score.

  Step 3   Welder reviews photos; sends structured quote:
           {
             material_type       : confirmed,
             estimated_weight_kg : N,
             material_cost       : N × current_rate_per_kg,
             labour_hours_est    : H,
             labour_cost_est     : H × labour_rate,
             total_lo            : material_cost + (labour_cost × 0.90),
             total_hi            : material_cost + (labour_cost × 1.10),
             site_visit_needed   : bool,
             visit_date          : (if needed)
           }
           material_cost and labour_cost ALWAYS shown separately in quote.

  Step 4   IF new_fabrication AND site_visit_needed:
             welder visits for measurement (visit is FREE; ₹50 token deducted later from invoice)
             welder uploads measurement_sketch_photo after visit
             final quote issued post-measurement

  Step 5   IF new_fabrication:
             design_approval step:
               welder uploads design_sketch or CAD reference photo
               user: APPROVE | REQUEST_CHANGES
               max 2 change cycles; beyond that, revised quote required
             ONCE APPROVED: fabrication begins; cutting starts

  Step 6   start_otp on day of work.
           Material use log:
             each stock piece cut → dimensions + weight photographed
           Structural weld joints photographed at mid-job.

  Step 7   Completion:
           structural_integrity_photo uploaded
             (clearly shows main load-bearing joints and weld bead)
           User physically tests item (open/close gate, press on railing, etc.)
           IF satisfied → end_otp → payment released
           IF structural concern raised → platform review; welder re-welds at no charge

  Step 8   Invoice:
           {
             item_type, job_type, material_type,
             material_kg_used, material_cost,
             labour_hours_actual, labour_rate, labour_total,

             hydrometer_readings[ ], load_test_result,
             distilled_water_ml, total
           }

1.14  MOSQUITO NET INSTALLATION
================================================================================

INPUTS:
  window_count  : integer 1–20
  windows[ ]    : each → { window_id, width_cm, height_cm,
                            window_type: sliding | casement | fixed }
  material      : fibreglass | aluminium | stainless_steel
  frame_type    : fixed | sliding | magnetic | velcro
  city_tier     : T1 | T2 | T3

MATERIAL RATE TABLE (₹ per sq.ft of net area, T3 before multiplier):
  material            T3_rate/sqft    description
  ─────────────────────────────────────────────────────────────────────
  fibreglass             ₹ 25         budget; light; standard durability
  aluminium              ₹ 45         sturdy; rust-resistant
  stainless_steel        ₹ 70         premium; long life; pet-resistant
  ─────────────────────────────────────────────────────────────────────
  final_rate = T3_rate × CITY_TIER_MULTIPLIER[city_tier]

INSTALLATION FEE PER WINDOW (T3):
  fixed      : ₹150
  sliding    : ₹200
  magnetic   : ₹250
  velcro     : ₹180

FRAME EXTRA PER WINDOW (T3):
  fixed      :  ₹  0
  sliding    :  ₹ 50
  magnetic   :  ₹ 80
  velcro     :  ₹ 30

AR MEASUREMENT (in-app camera feature):
  user points phone at window → edge-detection overlays W × H bounding box
  auto-populates width_cm + height_cm in booking form
  accuracy: ±2 cm  (user can manually adjust before confirming)

SQFT CALCULATION:
  area_sqft_per_window = (width_cm × height_cm) / 929.03
  total_sqft           = SUM(area_sqft for all windows)

QUOTE FORMULA:
  material_cost  = total_sqft × final_material_rate
  install_cost   = window_count × INSTALL_FEE[frame_type]  × CITY_MULT
  frame_extra    = window_count × FRAME_EXTRA[frame_type]  × CITY_MULT
  total_quote    = material_cost + install_cost + frame_extra
  deposit        = round(total_quote × 0.30)  (covers fabrication material cost)

ALGORITHM:
  Step 1   User enters window_count.
           OPTION A — AR mode: camera opened → auto-detect W × H per window.
           OPTION B — Manual: enter width_cm × height_cm per window.
           window_type per window recorded (affects frame compatibility).

  Step 2   Material selection:
           3 options displayed side-by-side:
             { material_name, rate_per_sqft, durability, weight, pet_safe, total_preview }
           user selects material.

  Step 3   Frame type selection:
           fixed    → permanent; cheapest
           sliding  → opens/closes; best for large windows
           magnetic → snap-on / snap-off; easy to remove seasonally
           velcro   → softest edge; easiest temporary option

  Step 4   Quote auto-calculated and itemised:
           FOR each window i:
             area_sqft[i] = (width_cm[i] × height_cm[i]) / 929.03
           total_sqft   = SUM(area_sqft)
           total_quote  = (total_sqft × final_material_rate)
                        + (window_count × INSTALL_FEE[frame_type] × CITY_MULT)
                        + (window_count × FRAME_EXTRA[frame_type] × CITY_MULT)
           itemised display:
             Material  (total_sqft sqft × ₹[rate])   : ₹[material_cost]
             Install   (window_count windows × ₹[fee]): ₹[install_cost]
             Frame     (window_count × ₹[extra])      : ₹[frame_extra]
             TOTAL                                     : ₹[total_quote]

  Step 5   Booking confirmed.
           deposit = 30% collected.
           fabrication_start = booking_date
           fabrication_TAT   = 2–3 working days
           installation_date = booking_date + 3 days (default)
           nets labelled by window number (W1, W2…) in workshop.

  Step 6   Installation day:
           each net fitted in sequence.
           AFTER each window:
             photo from inside + photo from outside uploaded.
             user checks: fit, no gaps at edges, smooth slide (if sliding frame).

  Step 7   User sign-off (in-app per window):
           FOR each window: CONFIRM FIT | FLAG ISSUE
           IF flagged: re-fabricate that net (1-day turnaround).
           WHEN all windows confirmed: balance payment collected + released.

  Step 8   Invoice:
           {
             windows[ ], material, frame_type, total_sqft,
             material_cost, install_cost, frame_extra,
             deposit_paid, balance_paid, total
           }

1.15  GLASS PANE REPLACEMENT
================================================================================

INPUTS:
  glass_type        : clear | frosted | tinted | toughened | wire_glass | one_way
  location          : window_pane | door_panel | tabletop | partition | showcase
  width_cm          : number
  height_cm         : number
  thickness         : 4mm | 5mm | 8mm | 12mm | toughened_8mm | toughened_12mm
  reference_photo   : image with coin or ruler in frame (mandatory; for scale)
  fragile_insurance : bool (opt-in ₹50 add-on; covers transit breakage)
  old_glass_removal : bool (default true; included in quote)

THICKNESS × MATERIAL PRICE TABLE (₹ per sq.ft, T3, material + cutting):
  thickness            T3 rate (₹)    typical use
  ─────────────────────────────────────────────────────────────────────────
  4mm clear                 60        small windows, photo frames
  5mm clear / frosted       80        standard windows and doors
  8mm clear                120        large windows, door panels
  12mm clear               180        structural panels, heavy doors
  toughened 8mm            200        bathrooms, shower enclosures, railings
  toughened 12mm           280        storefronts, heavy partitions
  ─────────────────────────────────────────────────────────────────────────
  glass_type add-on (per sqft, T3):
    frosted effect : +₹15    tinted effect : +₹20    wire inlay : +₹35
    one-way film   : +₹45    (add to base thickness price above)

INSTALLATION FEE (T3, based on pane area):
  area_sqft < 3     : ₹200
  3 ≤ area_sqft < 6 : ₹350
  area_sqft ≥ 6     : ₹500

FRAGILE INSURANCE:
  cost  : ₹50 flat add-on
  covers: accidental breakage of new pane during transit + installation
  claim : platform replaces pane at no extra charge within 24 hours

OLD GLASS DISPOSAL:
  included in installation_fee (labour for safe removal + recycling)
  disposal shown as "₹0 — included" in itemised invoice

SQFT CALCULATION:
  area_sqft = (width_cm × height_cm) / 929.03

QUOTE FORMULA:
  base_glass_rate  = THICKNESS_TABLE[thickness] × CITY_MULT
  type_addon       = GLASS_TYPE_ADDON[glass_type] × CITY_MULT
  material_cost    = area_sqft × (base_glass_rate + type_addon)
  install_fee      = INSTALL_FEE_TABLE[area_sqft_bracket] × CITY_MULT
  insurance        = 50  IF fragile_insurance ELSE 0
  total            = material_cost + install_fee + insurance

ALGORITHM:
  Step 1   User enters glass_type + location + width_cm + height_cm + thickness.
           area_sqft = (width_cm × height_cm) / 929.03
           live price auto-updates as any field changes.

  Step 2   Price auto-calculated (shown live, updates on every field change):
           material_cost = area_sqft × (base_glass_rate + type_addon) × CITY_MULT
           install_fee   = INSTALL_FEE_TABLE[bracket] × CITY_MULT
           display itemised breakdown + total.

  Step 3   Fragile insurance toggle (default OFF):
           IF ON: +₹50 shown; explained as "replacement guarantee if broken in transit"

  Step 4   Reference photo upload:
           mandatory (prevents wrong size being cut)
           user places coin or ruler in photo for scale reference
           IF photo missing: booking blocked

  Step 5   Glass shop query:
           platform queries GLASS_SHOP_DB:
             shops within 10 km WHERE glass_type IN stock AND thickness IN stock
           fastest_available_shop selected (stock + lead_time minimised)
           stock_status shown to user: "In stock at [shop_name], [X] km away"

  Step 6   Cutting + delivery scheduled:
           glass cut at shop to exact (width_cm × height_cm) ± 1 mm tolerance
           IF tolerance exceeded → recut before delivery
           delivery_window:
             booked before 12 pm → same-day delivery
             booked after 12 pm  → next-day delivery

  Step 7   Installation:
           old glass removed → before_photo (frame with broken glass documented)
           new glass fitted → sealed with putty / silicone / rubber bead
             (method depends on location and frame material)
           after_photo uploaded: new pane in situ, edges sealed

  Step 8   User inspection:
           check: no chips, no cracks, aligned within frame, secure when tapped
           IF damaged during transit AND insurance active:
             replacement triggered immediately; no extra charge
           IF satisfied → end_otp → balance payment released

  Step 9   Invoice:
           {
             glass_type, thickness, dimensions_cm, area_sqft,
             material_cost, glass_type_addon, install_fee,
             fragile_insurance_included (bool),
             old_glass_removed (bool), shop_name, total
           }
